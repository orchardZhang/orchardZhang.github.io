JAVA基础-数组与方法

2019-7-15

一维数组

  • 一维数组 : 使用数组必须经过 数组声明 和 分配储存空间两个步骤
  • 数组名是一个用户自定义的标识符，数组的维数用[]的个数来确定，
    类型标识符确定数组元素的数据类型，可以是基本类型也可以是引
    用类型如类（class）、接口（interface）等
  • 语法格式1
    类型标识符 数组名[]=null；
    或
    类型标识符[] 数组名=null；
  • 语法格式2 ：静态初始化数组
    类型标识符 数组名[]={初始值列表}；
  • 语法格式3 ;利用 new 关键字初始化数组
    类型标识符 数组名[]=null;//声明一个一维数组
    数组名=new 类型标识符[数组成都]；//利用new初始化一个数组
  • 语法格式4
    类型标识符 数组名[]=new 类型标识符[数组长度]；
  • 注意
    1 类型标识符 数组名[]=null，赋值为null表示还没有指向任何内存空间
    2 取得一维数组长度的方法时： 数组名称.length
    3 栈内存用来存数组名称，堆内存用来保存具体数据，使用new关键字来开辟内存空间

4.数组名可以直接代表一个地址值

  • 一维数组的引用格式： 数组名[数组下标] （从0开始）

二维数组及多维数组

  • 声明格式1
  • *数据类型 数组名[][]=null；**（推荐方式）
    数组名=new 数据类型[行的个数][列的个数]
  • 声明格式2
    数据类型 数组名[ ] [ ]=new 数据类型[行的个数][列的个数]；
  • 静态初始化数据类型 数组名[][]={各个元素};
  • score[i].length表示二维数组第i行数组元素的个数
  • 输出score[i]可以直接得到对应一维数组对应的数组地址，（即二维数组名配合索引可以得到一个一维数组，一维数组配合一个索引可以得到数组中的元素）

数组的初始化

  • 静态初始化：给出具体元素，数组长度由系统确定
    int[] arr = new int[]{1,2,3};
    或简化格式
    int arr = {1,2,3};
  • 动态初始化：给出数组长度，具体元素一般系统会直接以默认值给出，一般是0或false

数组常见问题

  • 素组索引越界异常：访问了不存在的索引,就是数组只有三个元素却访问了arr[3]
  • 空指针异常：数组已经不在指向堆内存的数据了，你还使用数组名去访问内存，例如arr=null（空常量），栈内存的数组名无法指向堆内存的元素值

方法

  • 方法，过程，函数时等价的
  • 方法定义
  • 实际参数：实际参与计算的参数
  • 形式参数：用于接收实际参数的参数
  • 方法如果是基本类型，形式参数的改变不影响实际参数
  • 方法如果是引用数据类型（地址值，指针）等于对实际参数的操作
    实际参数和形式参数两个引用同时指向同一个堆内，任何一方改变都会对堆内存产生影响

方法的重载

  • 在同一个类中，可以定义多个名字相同的方法，但方法的参数类型或参数的个数不同，与返回值无关
    编译器根据每个方法所用的参数类型个个数决定调用具体方法，可以采用强制转换的方法确定参数类型。
