JAVA基础-面向对象设计（1）

2019-7-19

对象和类

  • 类的定义
  • 类在某些方面可以看成是结构体struct的扩展
  • 对象=属性+方法（方法）
  • 对象的规范=属性定义+方法定义
  • 对象是一个变量（具体的东西）
    类就是类型（是规范，是定义），从万千对象中抽取共性，
    类规定了对象应该有的属性内容和方法，
    对象是类的具体实现
    例如：土豆丝菜谱是类，一盘土豆丝是对象
  • 三个特征
    继承性：子类可以继承父类的所用东西，但是对于父类的private成员，子类不能直接访问，必须通过父类的方法，子类可以扩展自己的方法
    封装性： 将客观世界中某个事物的属性和行为绑定在一起,具体表现就是将代码封装成方法，提高代码的复用性
    多态性：不同对象可以产生不同的结果
  • 基本类别的变量值很小，所以是直接值拷贝
    对象包含多个值，不容易复制，赋值采用共享同一块内存区域，所以是Reference赋值

对象的创建及使用

  • 创建对象
    类名 对象名表；//定义对象
    对象名=new 类名；//实例化对象

  • 或*
    类名 对象名=new类名（）；//声明对象的同时实例化对象（一般采用这种）

  • 数据成员引用格式
    对象名**.**数据成员名

  • 成员方法引用格式

  • 对象名**.**成员方法名（参数表）；

  • 重载overload 同一个类中，函数名称可以相同，但是函数参数的个数或者类型必须有所不同，不能以返回值来区分同名的函数

  • 类的成员变量（int、long、等）是有默认值的，而函数内部的局部变量是没有默认值的，需要初始化后才可以使用

构造函数

  • 构造函数的用处就是构造一个新建对象是提供的方法
  • 给对象的数据进行初始化
  • 构造函数的名称必须和类名一样（不能用VOID）且没有返回值类型和具体的返回值
  • Java有构造函数，但是没有析构函数，构造是制造对象的过程，析构时清除对象的过程
  • 每个变量都是有生命周期，它只能存储在离他最近的一对{}中
  • 变量创建时占用内存空间，变量消亡时回收内存空间，Java具有自动回收机制，JVM会自动回收所分配的对象的内存。
  • 每一个Java类都必须有构造函数
  • 如果没有显性定义的构造函数，Java编译器会自动为改类产生一个空的无形参构造函数
  • 如果我们提供了一个构造方法，系统将不提供空的无形参的构造函数
  • 一个类可以有多个构造函数，只要形参列表不相同即可
  • 在new对象的时候，根据实参的不同，自动挑选相应的构造函数，若匹配不上，将会报错

信息隐藏

  • 一般类的成员属性，是私有的private
    类的方法是共有public的，通过方法修改成员属性的值
  • 在main函数中，一般是通过类的方法来间接访问类的属性，而不是直接访问类的属性（信息隐藏）
  • 外界多类成员的操作只能通过getters和setters方法，可以利用Java IDE快速生成
    Source->generator getters and setters

private关键字

  • 通过对象直接访问成员变量，会存在数据安全问题
  • 权限修饰符，可以修饰成员变量和成员方法，被private修饰的成员只能在本类中被访问
  • 针对private修饰的变量，我们会提供get和set用于设置或获取成员变量的值

this指针

  • 表示当前类中的数据成员 this**.**数据成员名；
  • 调用本类的构造方法 this**.**参数列表；
  • 调用成员方法 this**.**成员方法名{参数列表}
  • 表示当前对象（一个地址）

static 关键字

  • static修饰成员变量和成员方法
    static 数据类型 数据成员名；
  • Static 可以被所有对象来共享，可以用类名调用，也就是说静态的对象不需要创建对象来调用
    （格式： 类名 . 静态对象或静态方法）
    静态的加载优先于对象
    随着类的加载而加载，加载在内存的方法区中
  • 静态成员的应用
    类名**.**静态成员；
  • 静态方法
    可以调用静态成员
    可以调用静态方法
    静态方法不能调用非静态成员变量，因为静态随着类的加载而加载，时间上优先于对象
    也不能调用非静态成员方法
  • 非静态方法即可以调静态也可以调用非静态的成员和方法
  • 也不能调用this
  • Math整个类都是satic方法和成员，为工具类

静态优缺点

  • 对对象的共享数据提供单独的存储，节省空间，没有必要每一个对象都存储一份
  • 可以直接被类名调用，不用再堆内存创建对象
  • 访问出现局限性，（静态只能访问静态）

封装

  • 封装原则：将不需要对外提供的内容隐藏，把属性隐藏，提供给公共方法对其访问
  • 提高代码的安全性和复用性

继承

  • 父类和子类的出现就是为了避免调用过多的方法
  • public class Man extend Human{ }；Human是父类，Man是子类，表示Man继承与Human
  • 子类也会继承父类的父类的属性和方法，以此类推
    比如，一个game父类有startgame 和 endgame 方法，则子类DotA和LOL就会继承父类，也有startgame和endgam的

继承中的成员变量特点

  • 继承只支持单继承，不支持多继承
    一个类只能有一个父类，不可以有多个父类,
  • Java支持多层继承
    C继承于B继承于A……
  • 子类只能获取父类中非private的成员
  • 子类父类成员变量的名字一样，就近原则
  • super：可以获取父类的成员变量，用法和this一致

继承中的成员方法特点

  • 方法的重写：在子父类当中，子类的方法和父类的方法完全一样，子类就重写了父类的方法，使用子类对象调用的方法就是子类的方法，
  • 子类中没有这个方法，就调用父类的
  • 使用场景 ：当父类的方法不能完全满足子类使用，这时候可以重写父类的方法，可以super调用父类的方法，这样可以保留父类的方法
  • 注解：@Override：表示重写父类方法
  • 如果父类成员权限是public，那么子类的权限要大于等于父类（只能为public）
  • 在有子类父类的继承关系中，创建子类的对象 ，调用子类的构造方法
    如果子类所有构造方法的第一行代码没有调用父类的构造方法，则默认的调用父类的无参构造；
    肯定（也必须）会先执行父类的构造，因为要先给父类的成员变量进行初始化，子类可能会使用到

    继承优缺点

  • 提高代码复用性和可维护性
  • 类的耦合性增强了，违反“高内聚低耦合” 的开发原则
    内聚：就是自己完成某件事情的能力
    耦合：类与类之间的关系

成员变量和局部变量的区别

  • 在类中的位置不同：成员变量在类中，方法外；局部类变量在方法中或方法声明上
  • 内存中的位置不同：成员便来那个在堆内存，局部变量在栈内存
  • 生命周期不同：成员变量随着对象的创建而存在，随着对象的消失而消失
    局部变量随着方法的调用而存在，随着方法的调用完毕而消失
  • 初始化值的不同：成员变量又默认值，局部变量没有默认值，使用时必须先赋值后使用
  • 如果有局部变量名和成员变量明相同，在局部使用时，采用就近原则

String类

  • 字符串的本质是一个字符数组

  • 构造方法

  • public String(String original)
    把字符串数据封装成字符串对象

  • public String(char[] value)
    把字符串数组的数据封装成字符串的对象

  • public String(char[] value, int offset，int count)
    把字符数组的一部分封装成字符串对象，offset索引、偏移地址，count个数

  • String s = “hello” 最简单最常用的

    判断功能

  • boolean equals（Object boj)
    比较字符串的内容是否相同，返回布尔值
    s1与s2比较

    忽略大小写比较字符串内容
    同上！

  • boolean startsWith(String str)
    判断字符串对象是否以指定的字符串开头

  • boolean endsWith(String str)
    判断字符串对象是否以指定的字符串结尾
    同上~

    获取功能

  • int length();
  • char charAt(int index);
    获取指定索引处的字符

对象数组

  • 对象数组是指数组的元素类型为对象类型，因为对象是引用类型默认值是null，因此使用对象数组是对每一个元素必须实例化，否则会出现引用异常；
  • 对象数组的定义
    类 对象数组名[]= new 类 {对象数组长度}
  • 对象数组里面的元素实际上是该元素对象的地址
    ##杂项
  • 引用类型的默认值是null
  • 一个标准类的组成包括：成员变量 构造方法 成员方法（获取成员变量）
  • 类名作为形式参数，则调用该方法需要的实际参数是作为形式参数的类的对象
  • 类名作为返回值，其实返回的是该类的对象
