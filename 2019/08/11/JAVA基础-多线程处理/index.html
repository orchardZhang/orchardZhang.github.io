JAVA基础-多线程处理

2019-8-11

多线程

  • 进程：一个应用程序在内存中的执行区域，一个进程可以有一个或者多个线程
  • 线程：线程依赖于进程；是进程的一个执行控制单元；执行路径
  • 单线程安全性高，但是效率低；多线程安全性低，但是效率高
  • 线程的生命周期： 新建->就绪（具备执行条件，没有执行权力）->运行（有执行条件和权力）->死亡
    还有一个过程的等待，用wait（）让线程等待，用notify（）将线程唤醒，然后进入就绪阶段等待cpu的指令。

多线程的实现方式

  • 创建一个新的执行线程有两种方法。 一个是将一个类1.声明为Thread的子类。 这个子类应该2.重写run类的方法Thread 3.创建对象 然后可以4.分配并启动子类的实例。要做四件事情
  • 主方法是单线程的，但是可以添加多线程的方法
  • 另一种方法来创建一个线程是声明实现Runnable接口。 那个类然后实现了run方法。 然后可以分配类的实例，在创建Thread时作为参数传递，并启动。
  • Thread(Runnable target)
    分配一个新的 Thread对象。
  • static Thread currentThread()
    返回当前的线程对象
  • 推荐使用这一种方式，因为这一种可以继续继承其他类，而上一种已经继承了Thread，由于java的单一继承，所以就不能再继承其他类了，而java是多实现接口的
  • 线程
  • static void sleep(long millis) 让当前线程停止，参数是毫秒数

synchronized 同步

  • 可以修饰代码块和方法，修饰的代码块和方法一旦被某个线程访问，其他线程就无法访问
  • 同步方法是一个安全的方法，一般被一个线程访问则全部锁死，其他线程无法访问
  • 非静态同步方法的锁对象是this，静态的同步方法的锁对象当前类的字节码对象
