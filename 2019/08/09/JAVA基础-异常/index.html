JAVA基础-异常

2019-08-09

异常

  • 异常：不正常，我们在写代码的时候编译或运行时出现的错误
  • 异常包含了错误的类型、原因、和出现的位置
  • 异常的体系结构
    throwable（最顶层）
    error（出现了不能处理的严重问题） exception（可以处理的一些问题）

异常的处理方式

jvm异常的处理方式

  • 如果出现异常我们没有处理，jvm会帮我们处理，将异常信息显示在命令行
  • 终止了程序，异常后的代码不在执行

捕获处理

  • try…catch
  • 执行顺序 try —->异常—->catch—>异常处理；
    如果没有出现异常就不会执行catch即异常处理

finally

  • 组合try/catch的,用于释放资源等收尾工作，无论怎样都会执行

抛出异常

  • 关键字throws
  • 当我们不想处理异常或者没有能力处理异常时就抛出，谁调用刚发睡就处理异常
  • 使用关键字throws在方法的声明处抛出异常
  • 当我们在主方法抛出异常，则是由jvm虚拟机来处理

处理多异常

  • 可以使用多个try..。catch语句
  • 使用一个try和多个catch，多个catch之间可以有子父类，同级之间没有顺序关系，如果存在子夫级的关系，父级必须得放在子级下面，否则就永远执行父级而不可能执行子级

throwable

  • String getMessage()
    返回该错误的原因。
  • String toString()
    返回异常的类型和原因
  • void printStackTrace()
    返回类型原因和位置
  • 编译器有快捷功能直接生成不用手动打入surround with —>try/catch Block

异常的分类

  • 运行时期异常RuntimeException：都是runtimeexception的子类就是运行时期异常，在编译时期可以选择处理或者不处理
  • 编译时期异常：是exception的子类，当不是runtimeexception类在编译时期必须处理，抛出或者捕获。

throw

  • 自定义异常
  • 制造异常并且结束方法
  • 如果扔出的是编译时期异常，必须在方法名中抛出
  • 怎么写自己的异常类呢？我们可以继承一个异常类RuntimeExcption或者Exception，然后实现多个构造就可以了，继承运行时期异常或者编译时期异常都可以，构造的实现可以通过工具快速实现
  • 利用自定义异常
